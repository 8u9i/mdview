// WHAT'S WORKING

Reading measure

72ch at 16px is the right zone for prose. Code blocks escape to 0.9em on a darker surface — clear visual separation.

Single accent

Teal only on the primary button, active outline item, brand mark, and inline code. 60-30-10 done correctly.

System stack

Segoe UI Variable on Win11 is a real choice, not a fallback. Avoids the Inter reflex that the smell detector flags.

Scroll-spy

Outline updates via rAF, not on every scroll event. Smooth and jank-free at any file size.

Keyboard path

Ctrl+O, Ctrl+B, Ctrl+=, Ctrl+-, Ctrl+0 are all wired with preventDefault. Tab order follows reading direction.

Drag-drop chain

Tauri webview event → DOM dragover → FileReader text fallback. Three layers, no broken paths.

// PRIORITY ISSUES
P1

Add a visible loading state

Between picking a file and rendering, the reader stays empty for 80-200ms with no feedback. The user clicks Open, sees the button react, then nothing — then content appears.

FIX: Wrap `elContent.innerHTML = clean` in a rAF, toggle `.loading` on `.prose` for 200ms with ease-out fade-in, or show a thin accent bar across the top of the reader.
P1

Empty state composition

The current empty state is a centered text block — the smell catalog's "center stack" reflex. The brief is Operate, not Decide. A centered hero implies a pitch; a viewer needs a workspace.

FIX: Replace the giant "MD" mark and centered headline with a 2-column layout: left = instructions + Open button, right = a small "Recent files" list or sample document. Or strip it down to a single line of copy + the primary action.
P2

M logo mark is a generic letter-tile

A rounded square containing the letter "M" is the AI-default brand mark. It works, but it does not commit. A more deliberate mark would represent the product — a document, a heading structure, a rendered line.

FIX: Replace the inner "M" with a small SVG showing 3 horizontal lines of varying length (a document outline). Drop the rounded-square container; place the mark on a sharper rectangle.
P2

Mobile / narrow layout

The outline sidebar has no mobile treatment. Below 800px, the titlebar's 3-column grid can overlap.

FIX: Below 800px, force the outline closed by default. Reduce titlebar `gap` to 8px and let `file-meta` ellipsis-shrink first. Drop the "Outline" label and keep the icon only.
P2

No error affordance in the reader

If `read_markdown` throws, the toast appears but the reader shows the previous content with no indication that the requested file failed.

FIX: On error, replace the reader content with an inline error block: file name, reason, "Try again" button. Keep the toast for transient feedback.
